Step by Step Sass

Lesson 1: Intro

  • Syntactically awesome style sheets
  • Pre-processor for CSS
  • 2 formats, .sass (older, original), scss (more similar to css)
  • Variables, Mixins and Extensions, Nesting, Import
  • Others including Less and Stylus

Lesson 2: Intalling Sass

  • Sass is part of Ruby
  • You need to install Ruby on systems where you compile Sass
  • Installed and ran via the command line (www.rubyinstaller.org)

Lesson 3: Class Project

  • Sass can be sprinkled in, but writing it from the start is where the power is.

Lesson 4: Compiling Sass

  • Run Sass from the cmd line
  • It watches a .scss file and automatically creates a .css file
  • Go to directory in cmd line > sass --watch style.scss:style.css

Lesson 5: Variables

  • Variables are declared with $
  • Variables can be hex value, string, int or boolean
  • Declare variables at the top (scope)
  • Use anywhere in the code where you want to replace
  • Compiling will look for other css issues and possibly throw errors

Lesson 6: Comments

  • Sass can use // to do single line comments that do not get outputted
  • Multiline still get outputted

Lesson 7: Nesting

  • Allows to nest selectors within other selectors (sub-properties (-family, -size, etc. can also be nested)
  • Don't nest at the BODY level
  • Try not to nest more than 3 levels
  • Benefits include less typing, organzies code, easier to maintain

Lesson 8: Extend

  • Essentially super variables
  • Allows you to re-use multiple lines of code
  • Define sets of properties that groups of elements could share
  • You can create a placeholder, it is base class that can have all the properties you need (e.g. %transition)
  • Then add @extend transition to your selector

Lesson 9: Mixins

  • Mixins should only be used for changing values
  • Use @extend to replicate multiple selectors
  • Mixins allow you to declare parameters/arguments
  • You can provide default values by adding :value after the parameter

Lesson 10: Parent Selector

  • Parent selector is represented by the &
  • You must nest your classes or IDs for it to work
  • Allows for psudeo classes to nested

Lesson 11: Import

  • Allows you break up your code into "partials" and then combines them back into one file, saves on server calls
  • A partial file starts with an underscore
  • Don't use an underscore when calliing
  • Import will look for .scss, or .css
  • You can also minimize your css using --style compress after your watch cmd (e.g. sass --watch style.scss:style.css --style compressed)

Lesson 12: Next Steps

  • You can also use functions and math
  • Things such as lighten or darken for colors
  • You can use the math functions to help with responsive websites
  • CMD is old fashioned, newer tools can help with the compiling
  • Compass is open-source CSS authoring toolkit, recommended (www.compass-style.org)
