Cross-Browser Compatible Websites with Bootstrap

Lesson 1: Installing Bootstrap

  • Use HTML5 doc type
  • Add lang="en" to HTML tag
  • Add meta tag utf-8
  • Force IE to use edge
  • Set device-wdith and initial scale

Lesson 2: The Responsive Grid

  • Do not override style in the Bootstrap file
  • 12 columns wide
  • 4 breakpoints (<768px, 768-921px, 922px-1199, >1200px)
  • You can nest grids
  • Rows delineate rows :)
  • Cols break at lg, md, sm, and xs
  • You can offset using

Lesson 3: Working with a Grid

  • You can rearrange using push and pull change the order of elements

Lesson 4: Typography

  • Bootstrap uses the Normalize.css
  • Helvitica Nueue is the default font
  • You can use class="h1" to match styles, but keep semmantics
  • class="lead" differentiates your starting paragraph
  • <s> - striketrough
  • <del> - strikethough for removed text
  • <u> - underline
  • <ins> - underline for inserted text
  • <mark> - highlight text
  • <small> - 85% of the current size
  • <strong> and <em> and similar
  • class="text-left", text-justify, etc. align, but do not float like standard alignments
  • class="text-lowercase", uppercase, etc.
  • <blockquote><p> have a cool style. Add <footer> for a cool credit
  • Lists have extra classes. class="list-unstyled" and list-inline
  • Definition Lists have a horizontal option

Lesson 5: Tables

  • class="table" adds standard formatting
  • table-striped, -hover, -bordered and -condensed add other features
  • most bootstrap elements can have class="active", danger, warning, success, info applied to them
  • class="table-responsive" - causes table to scroll on small devices. Must use FF hack at time of writing in an external CSS

Lesson 6: Forms

  • .form-control - adds simple styling
  • for="id name" will associate label with input
  • you can add .help-block in <p> for help
  • add helpers by adding a wrapping input <div class="input-group">
  • inside group add <span class="input-group-addon">$</span> for money
  • also add check boxes, radio button, additional buttons, etc to a group
  • You can make a form inline .form-inline
  • .sr-only for elements that only need to appear for screen reader
  • labels and inputs can appear side-by-side by adding the .form-horizontal to <form> and grouping entries in <div class="form-group>
  • Then add col classes (i.e. .col-sm-2) to the label and wrapping the input in a div with a complimentary col class
  • Checkboxes have -inline class option
  • Inputs and fieldsets can have disabled to diable them
  • Add .has-succes, .has-error to form-groups
  • You can add -lg to groups and input to make larger

Lesson 7: Commonly Used Components

  • Modals usually go at the bottom of the document
  • Just add .modal to identify a div as modal content
  • You can add a class of .fade for how it animates
  • .modal-dialog wraps .modal-content which wraps .modal-body all in <div>s
  • add data-toggle="modal" to the button to turn it on and data-target="#id" to target it
  • You can elements like .modal-header, -footer, etc
  • Add .alert and role="alert" to a <div> to create an Alert
  • You can also .alert-info, .alert-success
  • To make dismissable add .alert-dismissable and create a button like: <button type="button" class="close" data-dismiss="alert"><span aria-hidden="true">&times;</span><span class="sr-only">Close</span></button> (pay attention to accessibility)
  • Panels can also be a great way to add content as well, especially forms
  • You can .list-group to make interesting lists
  • Add .list-group-item to individual items
  • You can add badges by adding <span class="badge>value</span in side <li>
  • You can also use .list-group on <div>s. This can be used for list of links
